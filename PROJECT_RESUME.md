# 项目经历 — 智能差旅报销助手（MyAI-travel-reimbursement）

## 一、项目概述

基于 Spring Boot + AI 的企业差旅报销辅助系统。系统对接企业内部数据与微信文件，融合大语言模型（LLM）能力，实现发票 / 截图的智能 OCR 识别、报销材料自动整理、多维度 Excel 报表生成，以及全流程一键自动化报销。用户可通过自然语言与 AI 助手对话完成报销操作，大幅降低人工整理与核对成本。

---

## 二、核心技术栈

### 后端

- **语言 / 框架：** Java 21 + Spring Boot 3.x
- **ORM：** MyBatis-Plus
- **AI 框架：** LangChain4j（集成阿里通义千问等大模型）
- **数据库：** MySQL
- **文件存储：** 本地磁盘（可扩展至 OSS）
- **构建工具：** Maven
- **容器化：** Docker

### 前端（简要提及）

- 基于 Vite + Vue 3 构建的后台管理界面，负责项目、文件、AI 会话等模块的页面展示与交互。

> 注：前端部分由团队其他成员协作完成，本人在本项目中主要负责后端核心功能设计与实现。

---

## 三、主要功能模块

### 1. 项目与文件管理

- 支持多报销项目（项目维度）的独立管理，内置层级文件夹结构；
- 允许上传报销凭证（发票、截图、收据等），并关联至指定项目与目录；
- 提供文件的元信息维护与访问控制。

### 2. AI 智能助手（报销对话）

- 基于 LangChain4j 构建 AI 对话引擎，支持多轮上下文记忆；
- AI 可理解用户的自然语言报销意图，自动完成：
  - 发票内容 OCR 识别与结构化提取（报销人、金额、日期、发票类型等）；
  - 报销材料的智能分类与整理；
  - 报销规则与异常的智能提示。
- 支持 AI 会话历史的保存与回溯。

### 3. 一键自动化报销（核心亮点）

- 只需用户发出指令，AI 自动完成从凭证上传 → OCR 识别 → 数据整理 → Excel 报表生成的全链路闭环；
- 显著提升报销效率，减少人工操作与出错概率。

### 4. 报表生成与导出

- 基于 Apache POI 生成结构化 Excel 报销明细表；
- 支持按项目、时间范围等维度定制报表内容与格式；
- 提供手动触发与 AI 自动化两种触发方式。

### 5. 用户认证与权限

- 基于 JWT 的用户身份认证与接口鉴权；
- 支持用户注册、登录等基础账户管理功能。

## 四、数据库设计

系统设计了 9 张核心业务表，覆盖用户、项目、文件、文件夹、报销报表、AI 对话会话等实体，结构清晰，索引合理，具备良好的可扩展性。

## 五、项目亮点

| 亮点 | 说明 |
|------|------|
| **AI + 报销场景深度结合** | 将 LLM 能力落地到真实企业报销场景，实现从凭证识别到报表生成的全自动化闭环 |
| **自然语言驱动** | 用户通过自然语言即可完成报销操作，降低使用门槛，提升体验 |
| **可扩展的 OCR 架构** | 基于 LangChain4j 抽象接口，支持灵活切换不同 AI 服务提供商 |
| **清晰的分层架构** | Controller → Service → Mapper 分层清晰，代码易于维护与扩展 |
| **容器化部署** | 提供 Dockerfile 支持，便于快速部署与运维 |
